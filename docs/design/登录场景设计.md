# 登录场景设计（Login Scene）

> 状态：设计稿 v2（与 `src/game` 实际实现一致，登录成功已改为跳转主游戏场景）
> 范围：账号密码登录界面。本地写死凭据的 demo 阶段，登录成功经 `IGameFlow` 上报、由 Director 切换到主游戏场景；尚未接入服务器鉴权。

## 1. 需求

- 竖屏 720×1280 登录界面：标题、账号输入框、密码输入框、登录按钮、错误提示。
- 本地写死开发凭据 `admin / admin`，无注册、无数据库。
- 校验顺序：先判账号是否存在，再判密码是否正确，分别给不同错误提示。
- 密码密文显示；输入框带闪烁光标；支持回车提交、Tab 切换焦点。
- 鼠标与触摸均可操作；iOS 聚焦输入框时唤起系统软键盘。
- 自适应任意屏幕尺寸/方向（虚拟分辨率 + letterbox）。
- 登录成功跳转主游戏场景。

## 2. 分层与职责

| 层 | 角色 | 职责 |
| --- | --- | --- |
| Director | `Director` | 拥有窗口/主循环/资源管理器/平台路径；加载字体、按 `SceneId` 建场景、决定流转、letterbox 视口 |
| Scene | `LoginScene` | 实现 `IScene`，构造时装配并自持 `AccountModel` / `LoginView` / `LoginController` 并绑定；向 Director 暴露 `handleEvent/update/draw` |
| Controller | `LoginController` | 收 View 的登录上报 → 调 Model 校验 → 据结果回写 View；收 Model 的鉴权通知 → 经 `IGameFlow` 上报登录成功 |
| View | `LoginView` | 绘制全部 UI、采集鼠标/触摸/文本输入、密码掩码、光标闪烁、软键盘；经 `userEvent` 上报；被动 `draw` |
| Model | `AccountModel` | 运行期鉴权状态 + 凭据校验；状态变化经 observer 通知 |
| Config | `WindowConfig`、`ResourceConfig`、`AccountConfig`、`LoginUiConfig` | 窗口/字体/凭据/界面布局等静态具名常量 |
| Data | `AuthState`、`LoginRequest`、`LoginResult` | 无行为数据结构与枚举 |
| Delegate | `ILoginViewDelegate`、`IAccountObserver`、`IGameFlow`（共享） | View→Controller、Model→Controller、Controller→Director |
| Infra | `ResourceManager`、`PlatformPaths` | 字体缓存、平台路径（详见平台抽象层设计） |

## 3. 依赖方向

```
Director → 当前 IScene, ResourceManager, IPlatformPaths                            [允许依赖 SFML]
LoginScene → { AccountModel, LoginView, LoginController }                          [实现 IScene，允许依赖 SFML]
LoginController → LoginView, AccountModel, Delegate(含 IGameFlow), Config, Data    [纯逻辑，不依赖 SFML]
LoginView       → Config, Data, Delegate(ILoginViewDelegate), 资源管理器(字体), SFML
AccountModel    → Config, Data, Delegate(IAccountObserver)                         [纯逻辑，不依赖 SFML]
```

- 仅 `LoginView`、`LoginScene`、`Director` 及 Director 持有的 Infra 依赖 SFML；`LoginController` / `AccountModel` / `Config` / `Data` 一律不依赖 SFML。
- 下层经 Delegate 接口向上通信，不持有 Controller / Director 具体类型；Controller 经 `IGameFlow` 上报语义事件，不认识 `SceneId`、不自行切场景。

## 4. 接口（Delegate 层）

```cpp
// ILoginViewDelegate.h（View → Controller）
class ILoginViewDelegate {
public:
    virtual ~ILoginViewDelegate() = default;
    virtual void onLoginRequested(const LoginRequest& request) = 0;
};

// IAccountObserver.h（Model → Controller）
class IAccountObserver {
public:
    virtual ~IAccountObserver() = default;
    virtual void onAuthStateChanged(AuthState state) = 0;
};

// IGameFlow.h（Controller → Director）：onLoginSucceeded() / onQuitRequested()
```

## 5. 关键类

### AccountModel（Model）
- 持 `AuthState`（`LoggedOut` / `LoggedIn`）与 `currentAccountId`，维护 observer 列表。
- `checkLogin(username, password)`：先比对 `AccountConfig::kDevAccountName`，不符返回 `AccountNotFound`；再比对密码，不符返回 `WrongPassword`；否则 `Success`。纯校验、`const`、不改状态。
- `markLoggedIn(accountId)`：写账号标识、置 `LoggedIn`、通知 observer。
- 不感知 View，不依赖 SFML。

### LoginController（Controller）
- 持 `LoginView&`、`AccountModel&`、`IGameFlow&` 的非拥有引用。
- 用两个**内部适配器**接收上报，避免多重继承：
  - `ViewDelegate : ILoginViewDelegate` → 转发到 `_handleLoginRequested`。
  - `AccountObserver : IAccountObserver` → 转发到 `_handleAuthStateChanged`。
  - 经 `viewDelegate()` / `accountObserver()` 暴露给场景注册。
- `_handleLoginRequested`：调 `checkLogin`，`Success` → `markLoggedIn`；`AccountNotFound` / `WrongPassword` → `view.setErrorMessage(对应文案)`。
- `_handleAuthStateChanged`：状态为 `LoggedIn` 时 `_flow.onLoginSucceeded()`，由 Director 决定切到主场景。

### LoginView（View）
- 值成员持全部 SFML 对象：背景、账号/密码框、登录按钮、光标（`sf::RectangleShape`），标题/标签/输入/按钮/错误文本（`sf::Text`）。字体来自 ResourceManager（生命周期长于本 View）。
- 焦点状态 `InputField`（None/Account/Password），聚焦框高亮描边、唤起/收起软键盘。
- `handleEvent`：
  - 鼠标左键 / `TouchBegan` → `mapPixelToCoords` 换算后命中输入框聚焦、命中按钮提交、点空白失焦。
  - `TextEntered` → 退格码走删除、回车/换行码走提交、可见 ASCII 入字符串；密码以掩码 `*` 显示。
  - `KeyPressed` 仅处理桌面 Tab 切换焦点。
- `update(dt)`：按 `kCaretBlinkInterval` 翻转光标可见、更新光标位置（`findCharacterPos`）。
- `draw`：被动绘制当前状态。
- `setErrorMessage` 由 Controller 调用推入待显示数据。

### LoginScene（Scene）
- 实现 `IScene`，`unique_ptr` 自持 `AccountModel` / `LoginView` / `LoginController`。
- 构造时用 `ResourceManager` 按 key 取字体建 View，建 Controller 时注入 `IGameFlow&`；把 Controller 的 `viewDelegate()` 注册给 View、`accountObserver()` 注册给 Model。
- `handleEvent/update/draw` 转交内部 `LoginView`。

### Director（装配与驱动）
- `_loadUiFont`：用 `IPlatformPaths::resourceDir()` + `ResourceConfig` 相对路径加载字体到 ResourceManager，失败返回 `false` 终止。
- `_createScene(SceneId)`：工厂，按标识建 `LoginScene` / `MainScene`，传入 `ResourceManager&` 与内部 `GameFlow` 适配器（作 `IGameFlow`）。
- 主循环每帧：`_processEvents`（`Closed` 关窗、`Resized` 重算 letterbox、其余转交当前场景 `handleEvent`）→ `_update`（当前场景 `update`）→ `_render`（`clear` → 当前场景 `draw` → `display`）→ `_applyPendingSceneChange`（帧末执行待切换/退出）。

## 6. 时序

### 6.1 装配
1. Director `_loadUiFont` → `_createScene(Login)`。
2. LoginScene 建 Model/View/Controller，View 的 `userEvent` 委托 = `controller.viewDelegate()`，Model 的 observer = `controller.accountObserver()`。

### 6.2 登录提交与跳转
1. 用户点击/触摸登录按钮，或在输入框按回车。
2. `LoginView::_submit` 清错误提示，组 `LoginRequest`，经 `onLoginRequested` 上报。
3. `LoginController::_handleLoginRequested` 调 `AccountModel::checkLogin`。
4. 据 `LoginResult`：
   - `Success` → `markLoggedIn` → Model 通知 observer → Controller 收 `onAuthStateChanged(LoggedIn)` → `_flow.onLoginSucceeded()`。
   - `AccountNotFound` → `view.setErrorMessage("账号不存在")`。
   - `WrongPassword` → `view.setErrorMessage("密码错误")`。
5. `Director::onLoginSucceeded` 记录待切换场景 `Main`（不立即切）。
6. 本帧"轮询 → 更新 → 渲染"结束后，`_applyPendingSceneChange` 销毁登录场景、装配主游戏场景——避免在登录场景自身的事件回调中销毁它导致崩溃。

## 7. 跨平台与适配

- **指针**：桌面鼠标与移动端触摸（`TouchBegan` 单独处理，SFML 不把触摸映射为鼠标）统一走命中逻辑。
- **软键盘**：聚焦输入框 `setVirtualKeyboardVisible(true)`，失焦 `false`；桌面端为空操作，无平台分支侵入。
- **文本**：退格/回车统一经 `TextEntered`，兼容桌面与移动端软键盘。
- **屏幕适配**：UI 在 720×1280 虚拟坐标下设计，Director 用 `sf::View` + letterbox 等比缩放居中留边，输入经 `mapPixelToCoords` 反算虚拟坐标（详见平台抽象层设计第 11 节）。

## 8. 配置项（Config）

- `WindowConfig`：窗口 720×1280、标题。
- `ResourceConfig`：UI 字体 key `ui_default`、相对路径 `fonts/SourceHanSansSC-Regular.otf`（思源黑体，OFL）。
- `AccountConfig`：开发占位凭据 `admin/admin`、本地账号标识 `local-admin`（接服务器后删除）。
- `LoginUiConfig`：字号、布局坐标、描边、可见 ASCII 范围、退格/回车控制码、密码掩码字符、光标参数、全部 UTF-8 文案。
- 颜色为 View 内匿名命名空间的具名 `sf::Color`，不入 Config（避免 SFML 污染纯逻辑层）。

## 10. 现状与演进（诚实说明）

- **登录成功已跳转主游戏场景**（经 `IGameFlow.onLoginSucceeded()` 上报，由 Director 帧末切换），不再用弹窗。
- **凭据本地写死、无加密**，仅供 demo；真正鉴权待服务器阶段，届时 `checkLogin` 改为经服务接口异步校验，`AccountConfig` 占位常量删除。
- **验证边界**：macOS 本机可编译运行；Windows / iOS / Android 需在对应环境验证（见平台抽象层设计第 9 节）。
- **存档系统接入（计划）**：`IGameFlow.onLoginSucceeded()` 将改为携带账号 ID（`onLoginSucceeded(accountId)`），把登录身份传给主场景用于读档；并新增 `onArchiveOwnershipRejected()`——当读到的存档不属于当前登录账号时，退回登录场景、不加载任何数据。详见《存档系统设计》。
