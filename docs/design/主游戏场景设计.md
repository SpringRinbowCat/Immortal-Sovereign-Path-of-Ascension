# 主游戏场景设计（Main Scene）

> 状态：设计稿 v3（已实现,与 `src/game` 一致;场景经 Director 内部 `GameFlow` 适配器上报 `IGameFlow`）

## 1. 需求

- 登录成功后进入主游戏界面。
- 界面**只有左下角一个「退出游戏」按钮**,点击即退出;其余功能全部移除。
- 竖屏 720×1280,鼠标/触摸均可,自适应任意屏幕(沿用屏幕适配)。

## 2. 分层与职责

| 层 | 角色 | 职责 |
| --- | --- | --- |
| Scene | `MainScene` | 实现 `IScene`,构造时装配并自持 `MainView` / `MainController` 并绑定;向 Director 暴露 `handleEvent/update/draw` |
| Controller | `MainController` | 收 View 的退出上报,经 `IGameFlow.onQuitRequested()` 上报 Director |
| View | `MainView` | 画左下角退出按钮;采集鼠标/触摸命中;被动 `draw`;颜色为 View 内具名 `sf::Color`,背景用窗口清屏色 |
| Config | `MainUiConfig` | 退出按钮布局/字号/文案具名常量(竖屏 720×1280) |
| Delegate | `IMainViewDelegate`、`IGameFlow`(共享) | View→Controller、Controller→Director |

> 本场景无运行期数据,故无 Model、无 observer。

## 3. 依赖方向

```
MainScene → { MainView, MainController }   [实现 IScene，允许依赖 SFML]
MainController → Delegate(IMainViewDelegate / IGameFlow)        [纯逻辑，不依赖 SFML]
MainView       → Config(MainUiConfig), Delegate(IMainViewDelegate), SFML
```

- 仅 `MainView` 与 `MainScene` 依赖 SFML;`MainController` 不依赖 SFML、不持有 Director、不认识 `SceneId`。

## 4. 接口

```cpp
// IMainViewDelegate.h（View → Controller）
class IMainViewDelegate {
public:
    virtual ~IMainViewDelegate() = default;
    virtual void onQuitRequested() = 0;
};
// IGameFlow（Controller → Director）：onLoginSucceeded() / onQuitRequested()
```

## 5. 关键类

- **MainView**：值成员持退出按钮(`sf::RectangleShape` + `sf::Text`);`handleEvent` 处理鼠标左键与 `TouchBegan`,经 `mapPixelToCoords` 换算命中退出按钮即 `onQuitRequested`(用前判空);`draw` 只画按钮,背景留给窗口 `clear`。
- **MainController**：实现 `IMainViewDelegate`(经内部 `ViewDelegate` 适配器);`onQuitRequested` → `_flow.onQuitRequested()`。只持 `IGameFlow&`。

## 6. 时序

### 退出游戏
1. 点击/触摸退出按钮 → `MainView` 经 `onQuitRequested` 上报 Controller
2. `MainController`（内部 `ViewDelegate` 适配器）→ `_flow.onQuitRequested()`
3. Director 内部 `GameFlow` 适配器收到 → `_handleQuitRequested()` 置退出标志(不立即关)
4. 主循环帧末 `_applyPendingSceneChange()` → `_window.close()` → 主循环结束,程序退出

## 8. 演进（存档系统接入，计划）

存档系统接入后，本场景不再"无运行期数据"：

- `MainScene` 将自持 `PlayerModel`（玩家运行期数据真相），并接收 Director 注入的 `IArchiveStore&` 与当前登录账号 ID。
- `MainController` 增加存档读写协调：进场景异步读档并结算离线收益、运行期定时自动存、退出前补存；用内部适配器接收读写回调与 `PlayerModel` 的 observer 通知（沿用 `LoginController` 适配器模式）。
- 读档归属校验不符时，经 `IGameFlow.onArchiveOwnershipRejected()` 退回登录场景。
- 详见《存档系统设计》。本节为计划，未落地前 `src/game` 仍为"仅左下角退出按钮"的当前实现。
