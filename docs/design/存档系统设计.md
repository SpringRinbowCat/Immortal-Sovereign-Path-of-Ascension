# 存档系统设计（Save System）

> 状态：设计稿 v2（已实现，与 `src/game` 一致；完整需求/设计/任务见 `.kiro/specs/save-system/`）
> 范围：本地账号存档——单文件 JSON 持久化玩家资源与修炼成长，登录后异步读档并结算离线收益，运行期定时自动存、退出前补存。设备级设置文件、装备/词条、云存档与真正账号体系为范围外。

## 1. 需求（摘要，对应 R1–R12）

- 单文件 JSON 账号存档；原子写入（临时文件 + rename）防写坏。
- 存资源（铜钱/经验/修为）与成长（肉体等级/大境界/小层数/所选道）；攻/防/血不入档，运行期按公式现算。
- 异步读写、不阻塞主线程；完成回调经主线程派发器切回主线程，唯一交给 Controller。
- 运行期定时自动存 + 退出前补存；离线收益按离开时长结算、封顶 12 小时。
- 版本校验与迁移：低版本补默认升级；高版本拒绝并提示升级、不改文件。
- 轻量加密防随手改档（密钥不明文硬编码；真正防作弊靠后续服务器）。
- 健壮容错：缺失开新档；损坏则备份坏档 + 提示 + 开新档，不崩溃；字段非法回退默认。
- 平台可写目录经 `IPlatformPaths` 解析（见《平台抽象层设计》）。
- 账号归属校验：登录携带账号 ID，读档核对存档内账号 == 当前登录账号，不符则不加载、不改文件、退回登录。

## 2. 分层与职责

| 层 | 角色 | 职责 |
| --- | --- | --- |
| Delegate | `IArchiveStore` / `IArchiveCallback` | 异步 load/save 抽象与完成回调（唯一接收方 Controller） |
| Delegate | `IPlayerObserver` | 玩家数据变化通知 Controller |
| Delegate | `IGameFlow`（改造） | 新增携带账号 ID 的 `onLoginSucceeded(accountId)`、归属不符 `onArchiveOwnershipRejected()` |
| Service | `ArchiveService` | 本地实现：后台线程串行 load/save，编排原子写（临时文件 + rename）/损坏备份/路径解析；序列化与加解密委托下列两单元；纯标准库、无 SFML |
| Service | `ArchiveSerialization`（新增） | `PlayerArchiveData` ↔ JSON 互转（字段校验回退）与版本迁移/高版本判定；纯标准库、无 SFML |
| Service | `ArchiveCipher`（新增） | 轻量对称加解密与校验和（基于密钥流式 XOR），仅防随手改档；纯标准库、无 SFML |
| Infra | `Dispatcher` | 主线程派发器，线程安全任务队列，Director 每帧排空 |
| Config | `ArchiveConfig` / `ProgressionConfig` | 存档相对名/版本号/自动存间隔/离线上限；属性与收益基准常量 |
| Data | `PlayerArchiveData` / `CultivationTypes` / `ArchiveLoadOutcome` | 可序列化被动结构与共享枚举 |
| Model | `PlayerModel` | 运行期数据真相；按公式算攻/防/血；离线结算；observer 通知 |
| Controller | `MainController`（改造） | 发起读写、内部适配器收回调、自动存、更新 Model/View |
| Scene | `MainScene`（改造） | 自持 `PlayerModel`，接收注入的 `IArchiveStore&` 与账号 ID |
| Director | `Director`（改造） | 持 `Dispatcher`/`ArchiveService`；每帧排空；退出前存；登录账号 ID 传递与归属不符切回登录 |

## 3. 依赖方向

```
Director(SFML) ── 持有 ── Dispatcher(Infra) / ArchiveService(IArchiveStore) / IPlatformPaths
      └─ 注入 → MainScene ── 持有 ── PlayerModel / MainController / MainView
MainController → PlayerModel, IArchiveStore, IGameFlow, Data      纯逻辑,无 SFML
PlayerModel    → Data, Config, Delegate(IPlayerObserver)          纯逻辑,无 SFML
ArchiveService → Config, Data, Delegate(IArchiveStore/回调), IPlatformPaths, Dispatcher, nlohmann/json   纯标准库,无 SFML
Dispatcher     → 标准库(thread/mutex/queue)                       Infra,无 SFML
```

- 允许依赖 SFML 的只有 `Director` / `View` / `Scene`；`ArchiveService` / `PlayerModel` / `Config` / `Data` / `Dispatcher` / `MainController` 一律不依赖 SFML。

## 4. 接口（Delegate 层）

```cpp
// IArchiveStore.h（Controller → 持久化）
class IArchiveStore {
public:
    virtual ~IArchiveStore() = default;
    virtual void load(const std::string& accountId, IArchiveCallback& callback) = 0;
    virtual void save(const std::string& accountId, const PlayerArchiveData& data, IArchiveCallback& callback) = 0;
};

// IArchiveCallback.h（Service → Controller，已切回主线程）
class IArchiveCallback {
public:
    virtual ~IArchiveCallback() = default;
    virtual void onArchiveLoaded(const ArchiveLoadOutcome& outcome) = 0;
    virtual void onArchiveSaved(bool success) = 0;
};

// IPlayerObserver.h（Model → Controller）：onPlayerDataChanged()
// IGameFlow.h（改造）：onLoginSucceeded(const std::string& accountId) / onQuitRequested() / onArchiveOwnershipRejected()
```

`ArchiveLoadOutcome` 含状态 `ArchiveLoadStatus { Loaded, Created, RecoveredNew, RejectedTooNew, RejectedWrongOwner }` 与 `PlayerArchiveData`。

## 5. 关键类

- **ArchiveService（Service）**：实现 `IArchiveStore`。单后台 worker 线程串行执行 load/save（避免同一文件并发竞争）；序列化（`PlayerArchiveData` ↔ JSON、版本校验/迁移）与加解密分别由 `Service/Archive/` 下 `ArchiveSerialization`、`ArchiveCipher` 两个独立单元提供，ArchiveService 调用之；原子写（临时文件 + rename）、损坏改名备份、经 `IPlatformPaths::writableDataDir()` 解析路径并按需建目录为其内部私有方法；完成后 `Dispatcher::post` 把回调投回主线程；提供 `shutdownFlush()` 供退出时同步冲刷。纯标准库、不依赖 SFML。
- **Dispatcher（Infra）**：线程安全任务队列，`post`（任意线程）/`drain`（仅主线程每帧排空）。是"异步回调切回主线程"的统一载体。
- **PlayerModel（Model）**：运行期数据真相。`loadFrom`/`toArchiveData`、`settleOffline`（内部封顶）、`attack/defense/health`（现算：基础(肉体等级)+基础(小层数)+加成预留）、observer 通知。参照 `AccountModel` 的 observer 写法，纯 C++ 无 SFML。
- **MainController（改造）**：持 `PlayerModel&`/`IArchiveStore&`/`IGameFlow&` 与账号 ID；内部适配器实现 `IArchiveCallback` 与 `IPlayerObserver`（沿用 `LoginController` 模式）。进场景即 `load`；`onArchiveLoaded` 据状态处理（正常→loadFrom + 离线结算 + 推 View；`RejectedTooNew`→提示升级；`RecoveredNew`→提示损坏；`RejectedWrongOwner`→经 `onArchiveOwnershipRejected` 退回登录）；`update(dt)` 计时到点自动存。
- **Director（改造）**：持 `Dispatcher` 与 `ArchiveService`；主循环 `_processEvents` 后、`_update` 前 `_dispatcher.drain()`；`onLoginSucceeded(accountId)` 记录当前登录账号并注入主场景；`onArchiveOwnershipRejected()` → 帧末切回登录；`_pendingQuit` 时在 `_window.close()` 前 `shutdownFlush()`。

## 6. 数据模型

存档为分组嵌套 JSON；攻/防/血不入档；版本号置顶用于迁移；枚举以字符串持久化。

```json
{
  "version": 1,
  "accountId": "local-admin",
  "lastSaveTimeMs": 1782480000000,
  "resources": { "copperCoins": 12345, "experience": 6789, "cultivationPoints": 4200 },
  "growth": { "bodyLevel": 17, "majorRealm": "FoundationBuilding", "minorLevel": 3, "daoPath": "TianDao" }
}
```

## 7. 时序

- **登录后读档 + 离线结算**：登录成功（带账号 ID）→ 帧末切主场景 → `load` → worker 读/解密/反序列化/迁移/归属校验 → `dispatcher.post` → 主线程回调 → `loadFrom` + 离线结算 + 推 View。
- **自动存**：`update` 计时到点 → 置 `lastSaveTimeMs` → `save` → worker 序列化 + 加密 + 原子写 → `onArchiveSaved`。
- **退出存**：`onQuitRequested` → `_pendingQuit` → 帧末最终保存 + `shutdownFlush` → `close`。
- **损坏**：读档失败 → 不覆盖、坏档改名备份 → `RecoveredNew` → 提示 + 开新档。
- **版本**：低版本迁移补默认；高版本 `RejectedTooNew` → 提示升级、不改文件。
- **归属校验**：存档内账号 ≠ 登录账号 → `RejectedWrongOwner` → 不加载、不改文件 → `onArchiveOwnershipRejected` → 退回登录。

## 8. 数值与公式

攻/防/血 = 基础(肉体等级) + 基础(小层数) + 加成(预留 0)；基础值 = 基准 × 成长系数^(等级−1)，基准/系数由 Config 提供、Model 递推。离线收益 = 各资源每秒速率 × min(离线时长, 12 小时上限)，回拨按 0。具体数值待策划，Config 只给基准常量、不做判断。

## 9. 加密方案

明文 JSON → 轻量对称变换（如基于密钥的流式 XOR）+ 校验和 → 落盘；读取逆向并校验，篡改判损坏走容错。密钥不写成单个明文具名常量，由多段常量运行期派生。权衡：仅提高随手改档门槛，真正防作弊靠后续服务器。加解密集中在 `Service/Archive/ArchiveCipher` 单元，便于替换。

## 10. 线程模型

窗口/渲染仅主线程；`ArchiveService` 后台 worker 串行 I/O；回调经 `Dispatcher` 切回主线程唯一交 Controller。退出存 `shutdownFlush` 为同步例外。

## 11. 错误处理

| 情况 | 处理 |
| --- | --- |
| 存档不存在 | 默认值开新档并创建 |
| 损坏（解密/解析/版本校验失败） | 不覆盖、坏档改名备份、提示、开新档，不崩溃 |
| 数值字段为负 / 枚举非法 | 重置默认并继续 |
| 版本低于当前 | 迁移补默认升级 |
| 版本高于当前 | 拒绝、提示升级、不改文件 |
| 时钟回拨 | 离线时长按 0 |
| 写入中断（rename 前） | 保留上一份完整正式档 |
| 账号与登录不符 | 不返回数据、不改文件、退回登录 |
